# Design Doc: Secure FastAPI & Supabase Integration (SOLO-103)

## Goal
Secure the `POST /api/generate` endpoint using Supabase JWT and persist results directly to the Supabase database.

## Architecture
- **Auth Middleware:** A FastAPI dependency (`get_current_user`) extracts and verifies the `Authorization: Bearer <token>` header.
- **JWT Verification:** Uses `PyJWT` and the `SUPABASE_JWT_SECRET` (HS256).
- **Persistence:** A dedicated `persistence.py` module uses the Supabase service role key to bypass RLS and insert data on behalf of the verified user.
- **Graph Modification:** `GraphState` will include a `persist_locally` flag (default `True`). The `save_to_db_node` will skip the local SQLite write if `persist_locally` is `False`.
- **Error Handling:** SSE `error` events for invalid JWT, rate limits, and pipeline failures.

## Data Flow
1. **Request:** Client sends `POST /api/generate` with a Supabase JWT.
2. **Auth:** `get_current_user` verifies the JWT and extracts `user_id`.
3. **Pipeline:** LangGraph runs with `persist_locally=False`.
4. **Local Skip:** `save_to_db_node` does nothing.
5. **Supabase Save:** After the graph completes, the API handler calls `save_artifact_to_supabase`.
6. **Response:** SSE events stream progress, ending with a `complete` event containing the `artifact_id`.

## Dependencies
- `pyjwt[crypto]`: For JWT decoding.
- `supabase`: For database interaction (already installed).

## Security
- `SUPABASE_JWT_SECRET`: Used to verify user-sent tokens.
- `SUPABASE_SERVICE_ROLE_KEY`: Used to perform admin writes. **Must not be exposed to the client.**

## Testing
- Unit tests for JWT verification with mocked secret.
- Integration tests for the `generate` endpoint with mocked auth and Supabase client.
- Persistence tests using a mocked Supabase client.
