# SOLO-73 Design: Proactive Token Tracking and Pacing

## Overview
We will implement a client-side rolling window token tracker to prevent `429 Too Many Requests` errors from the Groq API. Since Groq's `llama-3.3-70b-versatile` has a strict free-tier limit of 6,000 Tokens Per Minute (TPM), we need to proactively halt our LangGraph pipeline execution when approaching this boundary.

## 1. TokenTracker Utility (`src/note_taker/llm.py`)
We will create a `TokenTracker` class to operate as a singleton or global module instance.
- **State:** A list or deque of `(timestamp: float, request_tokens: int)` pairs.
- **Methods:**
  - `add_usage(tokens: int)`: Appends current `time.time()` and token count to the state.
  - `clean_old()`: Iterates through the list and pops pairs that are older than 60 seconds.
  - `get_current_usage()`: Returns the sum of tokens in the remaining window list.
  - `wait_if_needed(estimated_next_tokens: int, max_limit: int)`: Calculates if adding `estimated_next_tokens` to `get_current_usage()` exceeds `max_limit`. If so, uses `time.sleep()` based on the timestamps in the list to wait exactly long enough for older requests to fall out of the 60-second window until the new request will fit.

## 2. Updated Invocation wrapper (`src/note_taker/llm.py`)
- We will update the `invoke_with_backoff` wrapper.
- **Before invocation:** The wrapper will call `wait_if_needed` (assuming an average chunk cost to be safe, e.g., 2000 tokens for the Draft step, or passed via a config parameter).
- **After invocation:** The wrapper will inspect `response.response_metadata['token_usage']['total_tokens']` and call `add_usage(actual_tokens)`.

## 3. Limits Configuration
- The limits will be hardcoded defaults within the `TokenTracker` class initially, specifically for `llama-3.3-70b-versatile` (6,000 TPM) and potentially other models if needed.
- We will expose these overrides as optional `kwargs` if we need flexibility later.

## Error Handling & Logging
- The `wait_if_needed` method should log a clear message to the CLI (e.g., `"Throttling to prevent 429: Sleeping for X seconds (Token usage: Y/6000)..."`) so the user knows the tool hasn't hung permanently.
